单调队列专题

Posted on 2023-01-09 In 题解 , 专题

POJ 2823 Sliding Window

Link

题意

对一个给定的序列，固定窗口大小为 k，让窗口从第一个元素开始，每次向后移动一个元素，因此对于长度为 n 的序列，总共可以得到 n-k+1 个窗口。求每次窗口内元素的最大值和最小值。

思路

下面以求最大值为例：

求窗口内最大值：维护一个单调递减的队列，队首元素为当前窗口内最大值。

  • 设窗口在 [l,r]，则队首元素是区间 [l,r] 的最大值，而队列中其他元素是后续窗口 [l+x, r+x] (x>0) 的可能的最大值（候选的最大值）。
  • 维护单调递减队列的过程中，如果队首元素不在当前窗口内，则弹出队首元素。
    • 队列中元素的出队时间有两种情况：
      • 新加入的元素比队尾元素大，队尾元素出队
      • 窗口中不再包含该元素
    • 如果队首元素一直没有因为新加入元素比它大而出队，那么队首元素一定是队列中最先出队的元素。
    • 因此，要保证队列中的元素都在当前窗口内，每次移动窗口时只需检查队首元素是否满足条件即可。因为窗口每次只移动一个元素。
  • 单调递减队列的维护过程：如果当前元素比队尾元素大，则不断弹出队尾元素直到当前元素比队尾元素小或者队列为空，然后才能将当前元素加入队尾。同时检查队首元素是否在当前窗口内，如果不在则弹出队首元素。

洛谷 P2698 [USACO12MAR] Flowerpot S

Link

题意

给定 xOy 第一象限平面上的 N 个水滴的高度，求最大高度差大于等于 D 的连续区间的长度的最小值。

注意数据范围，对应同一横坐标 x 可能有多个不同高度（不同 y）的水滴，并且 x 和 y 不受 N 限制。

思路

这道题相比于上一题的区别在于，区间的长度不是固定的，但也是求区间最大最小值。

很容易想到，对于横坐标为 x 的所有水滴，我们只需要存储它们的纵坐标 y 的最大值和最小值即可。因为只要花盆区间包含了 x，我们也只需要用到 y 的最大值和最小值。

然后就很好做了，就是对每个 x 对应的纵坐标的最大值最小值分别进行单调队列维护。但是动态变化的横坐标区间怎么解决呢？

可以先固定横坐标区间的左端点，而右端点不断向右扩展，直到区间内的纵坐标最大值最小值的差大于等于 D 为止。然后才将左端点向右移动一位。

洛谷 P2216 [HAOI2007] 理想的正方形

Link

题意

[HAOI2007]理想的正方形

题目描述

从一个 a*b 的矩阵中找出一个 n*n 的正方形区域，使得该区域所有数中的最大值和最小值的差最小。

思路

这题其实是个二维的单调队列，我一共维护了 4 个单调队列（Doge）。

我的做法是：

  • 先对 a 行分别计算每一行的区间最大值和区间最小值，区间长度固定为 n。
    • 第 1 行：列号构成的区间为 [1,n], [2,n+1], [3,n+2], ... , [b-n+1,b]，共 b-n+1 个区间。
    • ...直到第 a 行。
    • 得到 a 行的列区间最大值数组 mx 和最小值数组 mn。其中，mx[i][j] 表示第 i 行的第 j 个区间（即原数组 m 中的 m[i][j]~m[i][j+n-1]）的最大值，mn[i][j] 表示第 i 行的第 j 个区间的最小值。
  • 再对 b-n+1 个列区间的最大值（最小值）在行上维护单调队列（利用上一步得到的 mx 和 mn），区间长度固定为 n。
    • 第 1 列：行号构成的区间为 [1,n], [2,n+1], [3,n+2], ... , [a-n+1,a]，共 a-n+1 个区间。
    • ...直到第 b-n+1 列。
    • 得到 b-n+1 列的行区间最大值数组 rx 和最小值数组 rn。其中，rx[i][j] 表示第 i 列的第 j 个区间（即 mx 数组中的 mx[j][i]~mx[j+n-1][i], j=1,2,...,a-n+1）的最大值，rn[i][j] 表示第 i 列的第 j 个区间（即 mn 数组中的 mn[j][i]~mn[j+n-1][i], j=1,2,...,a-n+1）的最小值。
  • 最后得到的 rx 和 rn 数组中的每个元素就是一个 n*n 的正方形区域的最大值和最小值，作差取最小值即可。rx 和 rn 数组的大小都是 (b-n+1)*(a-n+1)。
    • rx[i][j] 表示第 i 列的第 j 个区间的最大值，rn[i][j] 表示第 i 列的第 j 个区间的最小值。

典型的贪心题.

这道题需要思考两点:

  • 能够成为出发点 start 的必要条件: gas[start] >= cost[start].
  • 如果从 start 出发, 到达某一点 i 时, 剩余的油量小于 0, 那么从 start 到 i 之间的任何一个点都不能作为出发点. 此时应考虑将出发点设为 i + 1.

第二条结论的证明:

  • 假设从 start 到 i 之间的某个点 j 能够作为出发点, 那么一定有 gas[start] >= cost[start] 和 gas[j] >= cost[j], 并且从 start 到 j - 1 的 gas[k] - cost[k] (k = start, start+1, ... j-1) 之和一定大于等于 0 (记这个和为 A, A >= 0). 而从 j 到 i 的 gas[k] - cost[k] (k = j, j+1, ... i) 之和一定小于 0 (记这个和为 B, B < 0), 并且 A 的绝对值小于 B 的绝对值. 因此, 减少 start 到 j - 1 这个序列只会让总油量减小而不会增大, 所以 j 不能作为出发点.

GitHub Personal Token

应用场景：我在配置 Hexo 的 deploy 部分时，不能使用原来的 https://github.com/[your username]/[your repo].git 或 git@github.com:[your username]/[your repo].git，而是要用 https://[your token]@github.com/[your username]/[your repo].git。

在博客根目录下的 _config.yml 文件中，deploy 部分的配置如下：

获得 Token 的具体操作

在 Setting - Developer settings 中就可以找到 Personal access tokens，点击 Generate new token，然后选择需要的权限，这里我只需要 repo 权限，然后点击 Generate token，就会生成一个 token，这个 token 就是你的 Personal access tokens，这个 token 只会显示一次，所以一定要保存好。

Developer settings 是 Setting 中的最后一项

我们的目的是想在本机终端实现 GitHub 用户身份的验证，以前需要在终端输入用户名和密码，但是这样的话很麻烦也不安全，现在已经用 Personal access tokens 取代了。

复制下该 token，于是从本机访问仓库的 URL 就变成了：https://[your token]@github.com/[your username]/[your repo].git，这样就可以实现身份验证并且顺利部署到页面了。
